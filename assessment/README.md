# Self-Assessment

This section helps you locate your platform on the five-stage sovereignty maturity scale.

## The five stages

| Stage | Name | Characteristic |
|---|---|---|
| 1 | **Erkennen** (Recognise) | Sovereignty issue identified, no concrete answer yet |
| 2 | **Erproben** (Pilot) | First sovereign workload, manual and inconsistent |
| 3 | **Einführen** (Operationalise) | First productive sovereign workload, repeatable pattern |
| 4 | **Skalieren** (Scale) | Multiple workloads, self-service, sovereignty as default |
| 5 | **Verankern** (Anchor) | Sovereignty is the path of least resistance, exit plan rehearsed |

## How to use

1. Read each stage description (in order).
2. Identify which stage best matches your current situation.
3. Use the *„Diagnostic markers"* sections to validate — most teams underestimate their stage.
4. Consult `axes-reference.md` for the five sovereignty axes that run orthogonal to the stages.

## Stages are not value judgements

A platform on Stage 2 (Erproben) is not *„immature"* — it *„is piloting"*. The stages are diagnostic, not aspirational. Some platforms hold Stage 2 productively for years because it matches their use-case profile. Whether to climb is itself a decision the framework helps you make.
