# The Five Sovereignty Axes

The five-stage scale runs *across* five orthogonal sovereignty axes. Each stage characterisation describes the state of all five axes for that stage.

| # | Axis | Question it answers |
|---|---|---|
| 1 | **Legal jurisdiction** | Under which legal framework does the platform operate? |
| 2 | **Operator residence** | Where is the operating entity registered, and where are its decision-makers? |
| 3 | **Data residency** | Where do data physically reside, and under whose legal access can they be subjected? |
| 4 | **Technical independence** | Can the stack be operated without inaccessible-by-design third-party dependencies? |
| 5 | **Exit capability** | Can you actually leave — has the exit path been documented and rehearsed? |

## Why five axes

A single sovereignty score is meaningless. *„EU-region"* marketing solves Axis 1 and Axis 3 partly while leaving Axis 2, 4, and 5 wide open. Real sovereignty diagnostics require all five.

## Per-stage axis status

Each stage's status across the five axes is described in the corresponding stage document.

## Source

The five-axes framework comes from Band 2 of the *Architecture Line* — *„Souverän statt hyperscaled — Die Souveränitäts-Disziplin für DACH-Plattformen"*. Band 4 (the companion to this framework) builds on these axes; this reference document is the operational distillation.
